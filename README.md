# Kontrollka PRO — Корпоративная система управления сетевым оборудованием

## 🚀 Отличия от Lite версии

| Функция | Lite | PRO |
|---------|------|-----|
| **Аутентификация** | Локальная (admin/admin) | Active Directory / LDAP + локальная |
| **Ролевая модель** | Нет (все admin) | Admin / Operator / Viewer |
| **Аудит действий** | Нет | Логирование всех действий пользователей |
| **Масштабирование** | До 50 устройств | 200+ устройств |
| **Кеширование** | Нет | Кеш устройств (TTL 5 мин) |
| **Прогресс-бар** | Нет | Для групповых команд с разбивкой на пакеты |
| **Ping-проверка** | Нет | Проверка доступности перед выполнением |
| **Ограничения** | Нет | Максимум 40 устройств за раз |
| **Пагинация** | Нет | Для истории и конфигураций |
| **Поиск** | По имени/IP | + поиск по содержимому конфигураций |
| **Импорт устройств** | Нет | Из CSV/Excel с предпросмотром |
| **HTTPS** | Нет | Гибкий режим с сертификатами |
| **HSTS** | Нет | Защита от downgrade атак |

## 🔐 Ролевая модель и маппинг групп AD

### Роли в системе:

| Роль | Права |
|------|-------|
| **Admin** | Полный доступ: управление устройствами, пользователями, настройками, выполнение любых команд |
| **Operator** | Выполнение команд, скриптов, просмотр конфигураций, но не может управлять пользователями и настройками |
| **Viewer** | Только просмотр: устройства, конфигурации, история команд (без выполнения) |

### Маппинг групп Active Directory:

В файле `.env` настраивается соответствие групп AD ролям:

```bash
# Группа администраторов (полный доступ)
AD_GROUP_ADMIN=CN=Network-Admins,OU=Groups,DC=company,DC=local

# Группа операторов (выполнение команд)
AD_GROUP_OPERATOR=CN=Network-Operators,OU=Groups,DC=company,DC=local

# Группа наблюдателей (только просмотр)
AD_GROUP_VIEWER=CN=Network-Viewers,OU=Groups,DC=company,DC=local
